ArcGIS Velocity Logger Help

Getting Started

Capture and log network data from TCP, UDP, HTTP/HTTPS, and gRPC connections. Useful for debugging and monitoring ArcGIS Velocity feeds and other data sources.

Connection Types

  • TCP Server: Listen for incoming TCP connections from clients
  • TCP Client: Connect to a remote TCP server to receive data
  • UDP Server: Listen for incoming UDP datagrams
  • UDP Client: Send UDP datagrams to a remote server
  • HTTP Client: Send data via HTTP/HTTPS POST requests to a remote endpoint (e.g. an ArcGIS Velocity HTTP Receiver feed). Supports JSON, Delimited (CSV), Esri JSON, GeoJSON, and XML formats.
  • HTTP Server: Start a local HTTP/HTTPS server that accepts POST requests from clients. Use for receiving data from the Velocity Simulator or other HTTP senders.
  • gRPC Server: Start a local gRPC server that accepts incoming RPC calls. Supports Protobuf, Kryo, and Text serialization.
  • gRPC Client: Connect to a remote gRPC server using HTTP/2. Supports Protobuf, Kryo, and Text serialization with client-streaming or unary RPCs.

HTTP Options

  • Format: Controls the Content-Type header — JSON (application/json), Delimited/CSV (text/plain), Esri JSON (application/json), GeoJSON (application/geo+json), or XML (application/xml). Must match the format configured in the ArcGIS Velocity HTTP Receiver feed.
  • TLS: Enable HTTPS (port 443 by default). Uses the OS certificate store automatically in client mode. Server mode requires a certificate and key.
  • CA cert path: Custom CA certificate file (PEM). Leave empty to use the OS certificate store. Only needed for enterprise or self-signed CAs.
  • TLS cert path: Client or server certificate file (PEM). Required for server-mode TLS; only needed in client mode for mutual TLS (mTLS).
  • TLS key path: Private key file (PEM). Required for server-mode TLS and client-side mTLS.
  • HTTP Path: URL path appended after host:port (default /). In server mode, only POST requests matching this path are accepted. In client mode, this path is used in outgoing POST URLs. Set to the Velocity feed's system-generated path (e.g. /receiver/feed-id) when connecting to a real endpoint.

gRPC Options

  • Serialization: Protobuf (default, external GrpcFeed protocol), Kryo (internal GrpcFeatureService, binary), or Text (internal GrpcFeatureService, UTF-8 CSV).
  • RPC Type: Client Streaming (persistent stream, higher throughput) or Unary (individual request/response per message).
  • TLS: Enable TLS for the gRPC connection. Uses the OS certificate store automatically.
  • Header Path Key / Path: Custom gRPC metadata header for endpoint routing (client mode only).

Log Management

  • Save Logs: Export captured data to text file
  • Clear Logs: Remove all log entries
  • Auto‑Scroll: Toggle automatic scrolling to newest lines from the header
  • List Order: Toggle between ascending (oldest first) and descending (newest first) order
  • Line Counter: Track total messages received

Status Bar and Indicators

The status bar at the bottom of the window provides real-time feedback on the application's connection state. It includes a colored dot indicator and descriptive text to show the current status.

Application States

Status Indicator Color Description
Connected ● Green A successful connection is established, and the application is ready to send or receive data.
Connecting ● Orange The application is attempting to establish a connection to the specified host and port.
Disconnecting ● Orange The application is in the process of closing the current connection.
Disconnected ● Grey The application is not connected. This is the default state before a connection is initiated or after a connection is successfully closed.
Error ● Red An error has occurred, such as a connection failure or configuration issue. Check the status message for more details.

Command Line Interface

The logger can run from the command line in normal UI mode or in a truly headless mode with no GUI. Press F3, or choose Help → Command Line Interface, or use the toolbar button (>_), or the context menu, to open a dedicated searchable dialog listing every command-line parameter, its default, whether it is required in headless mode, and an example.

  • Search + quick filters: filter by parameter text and switch among All, Required, Optional, Headless-only, and Help-related parameters.
  • Active filter pills: see the current search, category, and sort state at a glance and clear search/category filters directly from the pill row.
  • Copy / export tools: copy example commands, copy visible rows, or export visible rows as TSV, CSV, Markdown, or JSON.
  • Collapsible panels: the Behavior & Help Layouts and Notes sections start collapsed so the main table and examples stay in view.
  • Resizable table area: drag the parameter table taller or shorter inside the dialog as needed.
  • Wider first-open layout: the dialog opens wider by default so the built-in example commands are easier to read immediately.

For terminal help use npm run help:cli, npm run help:cli:wide, or npm run help:cli:narrow. See docs/COMMAND-LINE.md and docs/HEADLESS.md for full details.
